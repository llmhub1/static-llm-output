Using LLMs and File Sharing for Multi-Model Evaluation

You can set up a workflow to test multiple large language models (LLMs), publish their results, and control access to various types of files (text, audio, video, code, databases, archives, etc.) using a combination of secure file sharing platforms and LLM interfaces. Here’s how you can approach this:

1. File Sharing and Access Control

You cannot use me (Perplexity) directly as a file host or sharing platform, but you can leverage established secure file sharing services to manage and distribute your test materials and results. Leading options include:

  • Microsoft OneDrive for Business: Offers granular permissions, two-factor authentication, advanced audit logs, and integration with Office 365. You can set links to be public or restricted, and even set expiration for access.
  • Box Business: Supports integration with common productivity suites, granular permissions, encryption, and compliance controls. Suitable for organizations of any size.
  • Progress ShareFile (Citrix ShareFile): Enterprise-grade, supports a range of file types, advanced logging, encryption, and granular access. Allows for both cloud and on-premises deployment.
  • Kiteworks Secure File Sharing: Designed for highly regulated industries, supports a wide range of file types, extensive compliance, and customizable deployment (cloud, on-premises, hybrid). Offers advanced audit trails, policy-driven access, and supports large files up to 16 TB.
  • Thales SureDrop: Enterprise-class, defense-grade security, supports both cloud and on-premises deployment.
  • Egress Secure File Sharing: Focuses on secure collaboration and sharing with external organizations and end users.

These platforms allow you to:

  • Share files via public or private links.
  • Set permissions (view, edit, download).
  • Track file access and activity.
  • Integrate with common productivity and workflow tools.

2. LLMs for Evaluation and Comparison

While most LLMs (including Perplexity) do not natively host files, you can use them to:

  • Analyze, summarize, and compare the results stored on your chosen file sharing platform.
  • Generate reports or documentation based on the evaluation of LLM outputs.
  • Interface with APIs or tools (such as Wolfram suite, if needed) to process or visualize data.

For more advanced or automated workflows, you can:

  • Use open-source LLMs (like Llama, Mistral, or GPT derivatives) deployed locally or in the cloud, and connect them to your file storage via scripts or APIs.
  • Fine-tune LLMs on your own data and have them process or compare results as needed.
  • Develop custom tools to automate the evaluation and comparison process (leveraging your experience in tool development).

3. Out-of-the-Box and Integrative Solutions

No single LLM platform offers all-in-one secure file sharing and multi-model evaluation out of the box. However, you can combine:

  • A secure file sharing platform (for storage, access control, and collaboration).
  • LLM APIs or open-source LLMs (for analysis, summarization, and comparison).
  • Workflow automation tools (such as Zapier, Make, or custom scripts) to connect these components and automate processes.

4. Best Practices

  • Always use platforms with strong encryption (AES-256), multi-factor authentication, and audit trails for sensitive or private data.
  • Use granular permissions to separate public and private materials.
  • For LLM evaluation, ensure that your workflow can access and process all relevant file types (consider using conversion tools or APIs as needed).
